# Chat-With-PDF: A Multi-Approach RAG Playground 🚀

Welcome to **Chat-With-PDF**, a hands-on Retrieval-Augmented Generation (RAG) implementation designed to demonstrate and compare different search/retrieval architectures using the official **Google GenAI SDK**, **ChromaDB**, and **Gemini 2.5**.

This repository showcases three distinct approaches to building RAG systems:
1. **Manual In-Memory Similarity Search** (using NumPy and local JSON caching).
2. **Vector DB Persistent Search** (using ChromaDB for indexing and querying text files).
3. **PDF Document Chat** (extracting text from PDFs, chunking, vectorizing, and interactive querying with ChromaDB).

---

## 🛠️ Tech Stack & Key Technologies

- **LLM & Embeddings API**: [Google GenAI SDK](https://github.com/googleapis/google-genai-python) (`gemini-2.5-flash` for generation & `gemini-embedding-001` for vector embeddings).
- **Vector Database**: [ChromaDB](https://www.trychroma.com/) (Persistent database Client).
- **PDF Processing**: [PyPDF](https://pypi.org/project/pypdf/) (extracting text from PDF files).
- **Numerical Computation**: [NumPy](https://numpy.org/) (for manual cosine similarity calculations).
- **Environment Management**: `python-dotenv` (loading API keys securely).

## ⚙️ Setup & Installation

```bash
cd chat-with-pdf
```

### 2. Create a Virtual Environment & Activate It
```bash
python3 -m venv venv
source venv/bin/activate
```

### 3. Install Dependencies
```bash
pip install -r requirements.txt
```

Create a `.env` file in the root directory and add your Google Gemini API Key:
```env
GEMINI_API_KEY=your_gemini_api_key_here
```

## 🚀 How to Run

Choose any of the three RAG approaches to run and test:

### Approach 1: In-Memory Search (Manual Cosine Similarity)
Ideal for lightweight applications where a full database server or library setup is not desired.

1. Generate the embeddings cache:
   ```bash
   python index.py
   ```
   This reads `sample_document.txt` and exports `embeddings.json`.

2. Query the cached data:
   ```bash
   python main.py
   ```

---

### Approach 2: ChromaDB with Text
Uses ChromaDB's persistent client to index and query document embeddings.

1. Index the raw text file:
   ```bash
   python chroma_index.py
   ```

2. Query the indexed collection:
   ```bash
   python chroma_query.py
   ```

---

### Approach 3: Chat with PDF (Interactive Loop)
Extracts page-by-page text from `resume.pdf` (or any PDF of your choice renamed to `resume.pdf`), chunks it using a sliding window strategy, indexes it, and opens an interactive console loop to chat about the PDF contents.

1. Extract, chunk, and index the PDF:
   ```bash
   python pdf_index.py
   ```

2. Start the interactive chat terminal:
   ```bash
   python pdf_query.py
   ```
   *To exit the loop, type `quit` or `stop` in the prompt.*

## 🔍 How RAG Works Under the Hood (In this project)

1. **Document Loading**: Raw text is loaded directly or extracted from PDF pages via `PdfReader`.
2. **Chunking**: 
   - Paragraphs are separated by double newlines (`\n\n`) for basic text files.
   - For PDFs, a sliding window chunking strategy is used: `chunk_size = 1000` characters, with a step/overlap of `500` characters to maintain context continuity.
3. **Embedding Generation**: Text chunks are passed to the `gemini-embedding-001` model to output 768-dimensional dense vectors.
4. **Retrieval**:
   - *Manual*: The user's query vector is compared with each cached chunk vector using the dot product formula normalized by lengths (Cosine Similarity).
   - *ChromaDB*: The query vector is passed to `collection.query()`, retrieving the top 3 closest chunks.
5. **Generation**: The retrieved chunks are formatted as a context block, and combined with the user's question within a prompt. Gemini `gemini-2.5-flash` receives this prompt and synthesizes a precise answer. If no match is found, it safely responds with *"I don't have enough information."*
